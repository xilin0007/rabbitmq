生产者：发送消息
消费者：接收消息
交换器：生产者将消息发送到交换器，根据交换规则将消息放到队列中
	输入“”空字符串，表示使用默认的匿名交换器
	fanout （分发）：广播所有的消息，发布/订阅模式，生产者发布一条消息，所有的消费者（订阅者）都讲收到消息
	direct （直连）
		routingkey（消息路由）：生产者发送多个routingkey的消息，消费者只接收绑定了（关注）某个routingkey的消息，
	topic （主题）：
		routingkey（消息路由）：使用模糊匹配来绑定自己感兴趣的信息
		如果消费者端的路由关键字只使用【#】来匹配消息，在匹配【topic】模式下，它会变成一个分发【fanout】模式，接收所有消息
	headers （标题）
队列：存储消息的缓冲区

远程方法调用（RPC）
	服务端（生产者）实现好一个方法供客户端（消费者）远程调用
